# План перевірки гіпотез та Definition of Success

# План перевірки гіпотез
**Гіпотеза**: Користувачам потрібен простий інструмент для швидкої перевірки сумісності процесора та материнської плати перед покупкою.

**Перевірка**:
- Відстеження кількості успішних завершень головного сценарію (отримання повідомлення "Збірка сумісна").
- Збір стислих відгуків (оцінка від 1 до 5 або коротке опитування) після першого використання інтерфейсу.

# Невизначеності та їх зменшення
**Ризик:** Інтерфейс може бути незрозумілим для новачків.
- *Рішення:* Максимально спростити UI. Залишити лише два випадаючих списки без зайвих технічних характеристик або зробити окремий режим інтерфейсу для новачків/ентузіастів

# Артефакти між етапами
- **Вхідні артефакти:** Сформульовані вимоги у вигляді користувацьких сценаріїв.
- **Вихідні артефакти:** Мінімальний набір тестів приймання, готова до публікації попередня версія продукту.

# Definition of Success
MVP вважається успішним, якщо:
1. 8 з 10 нових користувачів можуть зібрати сумісну пару "Процесор + Материнська плата" менш ніж за 1 хвилину без підказок.
2. Жоден користувач під час тестування не зміг обійти систему і згенерувати несумісну збірку

# Наступні кроки (Після отримання перших результатів)
1. Проаналізувати відгуки перших користувачів щодо зручності інтерфейсу.
2. Додати можливість вибору оперативної пам'яті (RAM) та автоматичну перевірку її сумісності.
3. Дослідити можливість додавання орієнтовних цін на комплектуючі для розуміння загального бюджету збірки.
